说明 #

虚幻编辑器

了解碰撞 #

  1. 碰撞
    • 碰撞是世界中物体的物理交互
    • 如Character的Capsule组件, 因为它的存在, 我们跳起后会落到地面, 而不是穿过地板; 我们可以上楼梯, 或是和另一个Character碰撞(而不是穿过他)
  2. Logical Trace
    • 一条轨迹, 给定端点A和B, 计算与轨迹相交的场景中的物体
    • 射击时通过该判断, 得到击中目标

在虚幻编辑器使用Console #

  1. 鼠标已由程序控制, 打开Console
    按下 ` (apostrophe sign), 在视口下方会出现命令行, 可以输入debug命令
  2. 设置打开Console的快捷键
    项目设置 > Engine > Input > Console > Console Keys

输出每秒帧数 #

stat fps

在右上方角落显示, 一秒将近40帧, 40 x 25 ms = 1000 ms = 1 s



输出GPU统计数据 #

stat GPU



隐藏debug信息 #

输入相同命令, 或 

stat none

显示场景中碰撞相关 #

show collision

  • 可以看到Character的Capsule组件. 这很方便, 因为碰撞组件纯逻辑, 在游戏中默认不可见



  • 再次输入show collision关闭

碰撞选项 #

  1. 往场景中添加Cube和Sphere
    Place Actors > Shapes
  2. 碰撞选项

设置Character碰撞选项 #

设置CapsuleComponent的碰撞选项 #

BP_STUBaseCharacter > Capsule Component > Details > Collision

  1. 碰撞预设默认为Pawn



  2. 将碰撞预设置为Custom

设置Sphere的碰撞选项 #

  1. 碰撞预设选择Custom

  2. 允许游戏角色Overlap



  3. Character不可通过Cube, 但可通过Sphere

  4. Charater与Sphere重叠时, CameraComponent会有交叉反应-放大镜头, 因为CameraComponent处于Character中间, 这是由SpringArm造成的

查看SpringArm组件 #

BP_STUBaseCharacter > 选中SpringArmComponent > Details > Camera Collision > Probe Channel
SpringArm组件需要知道如何与世界进行物理交互

Probe Channel
Camera

我们可以修改 Sphere > Collision Responses > Trace Responses > Camera , 使得Character穿过Sphere时, CameraComponent忽略该事件

再次设置Sphere #

Collision Responses > Trace Responses > Camera, 选择Ignore



  • 之所以需要设置SpringArm的碰撞,因为当我们穿过墙时, 我们不应该可以看到墙的原先那一面
  • 现在我们穿过Sphere, 感官上正常

重叠事件: 输出Actor信息 #

设置事件 #

  1. 打开BP_STUBaseCharacter > Event Graph
  2. 在 Event ActorBeginOverlap 之后, 执行PrintString(在屏幕打印)
  3. OtherActor作为PrintString输入, 实际输出产生碰撞的物体名称



为Sphere设置产生重叠事件 #

  1. Sphere > Details > 勾选 Generate Overlap Events



    出于优化考虑, 默认false

  2. 日志



碰撞事件: 获取碰撞信息 #

设置Cube #

  1. 设置Collision Preset为Custom

  2. 勾选产生重叠事件
    Generate Overlap Events

  3. 允许Pawn重叠



获取碰撞信息 #

  1. 选择Cube, 勾选 Simulation Generates Hit Events



  2. BP_STUBaseCharacter > EventGraph, 添加EventHit

自定义碰撞相关 #

项目设置 > Engine > Collision

-
Object Channels 物体类型
Trace Channels
Preset 预设

自定义物体类型 #

Object Channels

创建物体类型Enemy #

  1. 点击New Object Channel



  2. 命名为Enemy, Default Response为Block



创建物体类型Geometry #

  1. 点击New Object Channel, 命名为Geometry

  2. 设置Default Response为Ignore



    其他类型自动忽略Geometry物体

查看Character对Geometry和Enemy的反馈 #

-
Enemy Block
Geometry Ignore



设置Cube #

  1. 物体类型为Geometry
  2. 忽略Pawn



设置Sphere #

  1. 物体类型为Enemy
  2. 阻挡Pawn



预设 #

可以自定义预设规则, 如A类型物体对其他类型物体如何作出反应; 或者修改已有预设
